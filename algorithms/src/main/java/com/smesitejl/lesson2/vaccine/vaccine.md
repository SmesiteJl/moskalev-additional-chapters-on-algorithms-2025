# Оптимизация производства вакцины в двух лабораториях

## 1. Постановка задачи
Имеются две лаборатории: **A** и **B**.  
Производство вакцины состоит из **n этапов**.

Для каждого этапа известны:

- время выполнения этапа в лаборатории A
- время выполнения этапа в лаборатории B
- время перехода A→B перед каждым этапом, начиная со 2-го
- время перехода B→A перед каждым этапом, начиная со 2-го

Требуется определить:

1. **минимальное время полного производства вакцины**
2. **траекторию производства** — на каком этапе выполнять операцию в A или B

---

## 2. Пояснение решения
Задача является вариантом классической задачи *Assembly Line Scheduling* и решается методом динамического программирования.

Вводятся две последовательности состояний:

- `TA[i]` — минимальное время производства до этапа *i*, если этап *i* выполняется в лаборатории A
- `TB[i]` — аналогично для лаборатории B

На каждом этапе рассматриваются два варианта:

- остаться в той же лаборатории
- перейти в другую лабораторию (с учётом стоимости перехода)

После заполнения DP-таблицы восстанавливается путь — траектория A/B по этапам.

---

## 3. Формат входных данных
Если этапов `n`, то ввод состоит из:

1. `n` — количество этапов
2. `n` чисел — время выполнения этапов в лаборатории A
3. `n` чисел — время выполнения этапов в лаборатории B
4. `n - 1` чисел — время перехода A→B (для этапов 2..n)
5. `n - 1` чисел — время перехода B→A (для этапов 2..n)

Всего: **4n - 2 чисел**.

---

## 4. Формат вывода
1. Минимальное возможное время производства
2. Последовательность A/B длины n
