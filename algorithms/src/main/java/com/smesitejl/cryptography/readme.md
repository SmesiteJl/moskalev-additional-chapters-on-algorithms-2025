# RSAExample — реализация алгоритма RSA (шифрование и цифровая подпись)

## 1. Постановка задачи
Требуется реализовать криптографическую систему RSA: сгенерировать пару ключей (открытый и закрытый), обеспечить шифрование и дешифрование сообщений, а также реализовать цифровую подпись и проверку подписи для произвольного сообщения.

---

## 2. Идея решения
Я использовал классическую схему RSA, основанную на трудности факторизации больших чисел. Сообщение переводится в набор байтов, разбивается на блоки подходящего размера, после чего каждый блок шифруется и расшифровывается с использованием модульной экспоненты.

---

## 3. Выбор алгоритма
Алгоритм RSA является стандартом асимметричной криптографии и естественным выбором для данной задачи. Для работы с большими числами используется тип BigInteger, который обеспечивает корректность вычислений при больших разрядах ключей.

---

## 4. Алгоритм в общем виде
1. Сгенерировать два больших простых числа p и q.
2. Вычислить модуль n = p · q и функцию Эйлера φ(n).
3. Выбрать открытую экспоненту e и вычислить закрытую экспоненту d.
4. Разбить сообщение на блоки.
5. Для каждого блока выполнить шифрование по формуле c = m^e mod n.
6. Для расшифрования выполнить m = c^d mod n.
7. Для подписи использовать возведение в степень d, для проверки — в степень e.

---

## 5. Корректность
Корректность алгоритма обеспечивается свойствами модульной арифметики и теоремой Эйлера. Поскольку e и d выбраны взаимно обратными по модулю φ(n), операция расшифрования полностью восстанавливает исходное сообщение. Аналогично, корректно работает проверка цифровой подписи.

---

## 6. Асимптотическая сложность
Пусть L — длина модуля n в битах.  
Основные операции — модульные возведения в степень.  
Их асимптотическая сложность определяется алгоритмом быстрого возведения в степень и составляет O(L³) в худшем случае для BigInteger.  
Память: O(L).

---

## 7. Особые случаи
- Сообщения произвольной длины обрабатываются разбиением на блоки.
- Корректно обрабатываются ведущие нули в байтовых представлениях.
- Поддерживается генерация ключей большой разрядности.
- Учитывается возможность пустого сообщения.

---

## 8. Связь с теорией
Алгоритм RSA основан на трудности факторизации больших чисел и теореме Эйлера. Используются свойства взаимной простоты, обратимости по модулю и быстрого возведения в степень, изучаемые в курсе теории чисел и криптографии.

---

## 9. Что реализовано в программе
- Генерация пары ключей RSA.
- Шифрование сообщения с использованием открытого ключа.
- Расшифрование сообщения с использованием закрытого ключа.
- Формирование цифровой подписи.
- Проверка подлинности подписи.
- Работа с сообщениями произвольной длины через разбиение на блоки.
