# Matchmaker — задача о свахе (двудольное паросочетание)

## 1. Постановка задачи
Задан набор невест с приданым и набор женихов. Известны допустимые пары. Требуется выбрать непересекающееся множество пар так, чтобы суммарное приданое выбранных невест было максимальным.

---

## 2. Идея решения
Я представил задачу в виде двудольного графа, где одной долей являются невесты, другой — женихи, а допустимые пары задаются рёбрами. Требуется найти паросочетание максимального веса.

---

## 3. Выбор алгоритма
Для поиска паросочетания используется алгоритм с увеличивающими цепями. Невесты перебираются в порядке убывания приданого, что позволяет максимизировать суммарный выигрыш.

---

## 4. Алгоритм в общем виде
1. Считать все приданые и допустимые пары.
2. Отсортировать невест по убыванию приданого.
3. Последовательно пытаться добавить каждую невесту в паросочетание через поиск увеличивающей цепи.
4. После обработки всех невест вычислить суммарное приданое.

---

## 5. Корректность
Алгоритм поддерживает корректное паросочетание на каждом шаге. Использование увеличивающих цепей гарантирует максимальное возможное число допустимых пар, а порядок перебора по приданому обеспечивает максимизацию суммарного веса.

---

## 6. Асимптотическая сложность
Пусть n — число невест, m — число женихов, E — число допустимых пар.  
Время: O(n · E).  
Память: O(n + m + E).

---

## 7. Особые случаи
- Отсутствие допустимых пар.
- Несколько невест с одинаковым приданым.
- Несколько оптимальных решений.
- Полное двудольное соответствие.

---

## 8. Связь с теорией
Используется классическая схема поиска максимального двудольного паросочетания с помощью увеличивающих цепей.

---

## 9. Что реализовано в программе
- Построение двудольного графа.
- Сортировка невест по приданому.
- Поиск увеличивающих цепей.
- Получение максимального по сумме приданого паросочетания.
