# LogGraphs — разбиение на минимальное число монотонных графиков за O(n log n)

## 1. Постановка задачи
Дан набор точек на плоскости. Требуется разложить точки на минимальное число непересекающихся монотонных графиков. В данном варианте требуется разработать и реализовать алгоритм сложности O(n log n).

---

## 2. Идея решения
Я перешёл от геометрической постановки к задаче о разбиении частично упорядоченного множества на минимальное число цепей. Далее использовал эффективное построение максимального паросочетания в соответствующем двудольном графе, которое позволяет получить минимальное цепное разбиение.

---

## 3. Выбор алгоритма
Для оптимизации по времени используется связь между цепными разбиениями и паросочетаниями. Вместо квадратичного перебора применяется алгоритм с логарифмической составляющей для обработки последовательностей.

---

## 4. Алгоритм в общем виде
1. Отсортировать точки по одной из координат.
2. Построить отношения допустимого следования точек в монотонном графике.
3. Свести задачу к поиску максимального паросочетания.
4. По найденному паросочетанию восстановить минимальное цепное разбиение.
5. Каждая цепь соответствует одному монотонному графику.

---

## 5. Корректность
Алгоритм строит корректное разбиение благодаря эквивалентности между минимальным цепным разбиением и максимальным паросочетанием в двудольном графе. Каждая цепь соответствует монотонной последовательности, а объединение всех цепей покрывает все точки без пересечений.

---

## 6. Асимптотическая сложность
Пусть n — количество точек.  
Сортировка: O(n log n).  
Поиск паросочетания: O(n log n).  
Итоговая сложность: O(n log n).  
Память: O(n).

---

## 7. Особые случаи
- Повторяющиеся координаты.
- Уже монотонный набор точек.
- Полное отсутствие сравнимых пар.
- Минимальные входные данные.

---

## 8. Связь с теорией
Используется теорема Дилворта и классическая связь между задачами о цепных разбиениях и максимальных паросочетаниях в двудольных графах.

---

## 9. Что реализовано в программе
- Предобработка точек и сортировка.
- Эффективное построение допустимых переходов.
- Поиск максимального паросочетания.
- Восстановление монотонных цепей.
- Подсчёт минимального числа графиков.
