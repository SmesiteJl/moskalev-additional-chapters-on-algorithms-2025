# VaccineProduction — задача о производстве вакцины

## 1. Постановка задачи
Задана производственная цепочка изготовления вакцины, разбитая на несколько последовательных этапов. Для каждого этапа известны трудозатраты и условия выполнения. Требуется минимизировать суммарное время или стоимость производства всей партии.

---

## 2. Идея решения
Я свёл задачу к динамическому программированию по этапам производства, где каждое состояние описывает минимальные затраты на выполнение первых i этапов при фиксированных условиях.

---

## 3. Выбор алгоритма
Задача обладает оптимальной подструктурой: оптимальное решение для всей цепочки строится на основе оптимальных решений для её частей. Это делает динамическое программирование наиболее подходящим методом.

---

## 4. Алгоритм в общем виде
1. Считать параметры всех этапов производства.
2. Построить таблицу dp, где dp[i] — минимальные затраты для первых i этапов.
3. Для каждого этапа перебрать допустимые варианты переходов.
4. Вычислить dp для последнего этапа и вывести результат.

---

## 5. Корректность
Каждое состояние dp[i] корректно отражает минимальные затраты для первых i этапов. Переходы учитывают все допустимые варианты выполнения очередного этапа, поэтому итоговое значение dp является оптимальным.

---

## 6. Асимптотическая сложность
Пусть n — количество этапов производства.  
Время: O(n) или O(n · k) в зависимости от числа вариантов переходов.  
Память: O(n).

---

## 7. Особые случаи
- Минимальное число этапов.
- Равные трудозатраты на этапах.
- Нулевые затраты.
- Большие значения входных параметров.

---

## 8. Связь с теорией
Используется классическая схема динамического программирования по стадиям, аналогичная задачам о конвейерной обработке и оптимальном разбиении.

---

## 9. Что реализовано в программе
- Чтение параметров этапов производства.
- Построение и заполнение массива динамического программирования.
- Вычисление минимальных затрат производства.
- Вывод итогового значения.
