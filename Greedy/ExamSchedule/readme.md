# ExamSchedule — задача о зачётной сессии (жадный алгоритм + DSU)

## 1. Постановка задачи
Задано n зачетов, для каждого известен день сдачи и штраф за просрочку. За одну ночь можно подготовиться только к одному зачету. Необходимо выбрать порядок подготовки так, чтобы суммарный штраф за несданные в срок зачеты был минимальным.

---

## 2. Идея решения
Я отсортировал все зачеты по убыванию штрафа и пытался каждому зачету назначить самый поздний возможный свободный день перед его дедлайном. Таким образом, наиболее «дорогие» зачеты стараемся сдать в первую очередь.

---

## 3. Выбор алгоритма
Жадный алгоритм здесь является естественным выбором, так как локально оптимальное решение — сдавать зачет с максимальным штрафом как можно позже — приводит к глобально оптимальному результату. Для быстрого поиска свободного дня используется система непересекающихся множеств (DSU).

---

## 4. Алгоритм в общем виде
1. Считать все зачеты.
2. Отсортировать их по убыванию штрафа.
3. Для каждого зачета попытаться назначить его на самый поздний свободный день не позже дедлайна.
4. Если свободного дня нет — добавить его штраф к суммарному.
5. Повторять до обработки всех зачетов.

---

## 5. Корректность
Алгоритм всегда в первую очередь размещает зачет с максимальным штрафом на максимально возможный свободный день. Это гарантирует, что если какой-либо зачет невозможно разместить, то это наиболее «дешёвый» возможный пропуск, а значит суммарный штраф минимален.

---

## 6. Асимптотическая сложность
Пусть n — количество зачетов, k — максимальный день дедлайна.  
Сортировка: O(n log n).  
Операции DSU: O(n α(k)).  
Итоговая сложность: O(n log n).  
Память: O(k).

---

## 7. Особые случаи
- Несколько зачетов с одинаковым дедлайном.
- Зачеты с одинаковыми штрафами.
- Случай, когда все дни заняты.
- Минимальные значения входных данных.

---

## 8. Связь с теорией
Используется классический жадный алгоритм для задачи планирования с дедлайнами и штрафами, а также структура данных DSU для ускорения поиска свободного дня.

---

## 9. Что реализовано в программе
- Сортировка зачетов по штрафу.
- Структура непересекающихся множеств для хранения свободных дней.
- Жадное назначение зачетов на допустимые дни.
- Подсчёт минимального суммарного штрафа.
