# MoscowPiterHappyTickets — счастливые билеты (московское и питерское определения)

## 1. Постановка задачи
Требуется по заданному диапазону билетов определить количество счастливых билетов по двум определениям:
- Московское: сумма первой половины цифр равна сумме второй.
- Питерское: сумма чётных цифр равна сумме нечётных.

---

## 2. Идея решения
Я разбрал число на цифры и отдельно вычислял суммы по требуемым критериям. Для московского варианта задействованы два подмассива цифр, для питерского — разделение по индексу.

---

## 3. Выбор алгоритма
Для каждого билета выполняется простой линейный проход по цифрам. Более эффективные методы здесь не требуются.

---

## 4. Алгоритм в общем виде
1. Разобрать число на цифры.
2. Для московского определения:
    - суммировать первую половину;
    - суммировать вторую половину.
3. Для питерского:
    - суммировать цифры на чётных позициях;
    - суммировать цифры на нечётных позициях.
4. Сравнить суммы.

---

## 5. Корректность
Каждая цифра учитывается строго в своей позиции, и суммы вычисляются по определению. Сравнение двух сумм однозначно определяет счастливость билета по каждому критерию.

---

## 6. Асимптотическая сложность
Время: O(n) для одной проверки.  
Память: O(1).

---

## 7. Особые случаи
- Нули в начале номера.
- Чётное / нечётное количество цифр.
- Комбинации с одинаковыми суммами.

---

## 8. Связь с теорией
Используются методы теории чисел и анализа цифровых последовательностей.

---

## 9. Что реализовано в программе
- Разбор билета на цифры.
- Проверка счастливости по московскому критерию.
- Проверка по питерскому критерию.
- Подсчёт количества счастливых билетов в диапазоне.
