# HarmonicLowerBound — оценка снизу для гармонического ряда

## 1. Постановка задачи
Требуется экспериментально или аналитически исследовать нижнюю оценку для частичных сумм гармонического ряда и сравнить её с теоретическими оценками.

---

## 2. Идея решения
Я вычислял значения частичных сумм гармонического ряда для различных n и сравнивал их с известными логарифмическими оценками.

---

## 3. Выбор алгоритма
Задача не требует сложных алгоритмов — достаточно последовательного суммирования значений и анализа полученных результатов.

---

## 4. Алгоритм в общем виде
1. Последовательно вычислять частичные суммы гармонического ряда.
2. Для каждого значения n сравнивать сумму с теоретической оценкой.
3. Вывести или сохранить результаты вычислений.

---

## 5. Корректность
Каждая частичная сумма вычисляется строго по определению гармонического ряда, поэтому полученные значения являются точными в рамках машинной арифметики.

---

## 6. Асимптотическая сложность
Пусть n — число членов ряда.  
Время: O(n).  
Память: O(1).

---

## 7. Особые случаи
- Малые значения n.
- Большие значения n, влияющие на точность.
- Погрешности вычислений.

---

## 8. Связь с теорией
Используется связь гармонического ряда с логарифмической функцией и асимптотическими оценками из математического анализа.

---

## 9. Что реализовано в программе
- Последовательное вычисление частичных сумм.
- Сравнение с теоретической оценкой.
- Вывод результатов эксперимента.
